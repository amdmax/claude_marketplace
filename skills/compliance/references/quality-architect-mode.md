# Quality Architect Mode (Phase 0)

> **Reference for:** compliance skill
> **Context:** AC generation, test linking, and quality gates

## Overview

Phase 0 transforms the compliance skill from a static validator into a **proactive quality architect** that enforces test coverage for acceptance criteria before commits.

**Workflow:**
```
Active Story → Extract/Generate ACs → Link to Tests → Validate P0 Coverage → Block Commit if Gaps
```

## Phase 0 Workflow

### Step 1: Read Active Story

```typescript
const story = JSON.parse(await readFile('.agile-dev-team/active-story.json'));

// Expected structure:
{
  issueNumber: 305,
  title: "Add JSON linting configuration",
  body: "...",
  url: "https://github.com/.../issues/305",
  nfrs: { performance: {...}, security: {...} },
  context: { relatedCode: [{path: "..."}] }
}
```

### Step 2: Extract/Generate Acceptance Criteria

**Strategy:** Hybrid approach

1. **Parse from story.body** (if exists):
   ```markdown
   ## Acceptance Criteria

   **Must Have (P0):**
   - [ ] JSON files validated with .jsonlintrc rules
   - [ ] Validation runs on pre-commit hook

   **Should Have (P1):**
   - [ ] Custom error messages for common issues
   ```

2. **Generate from title** (if missing):
   ```typescript
   {
     id: "AC-001",
     priority: "P0",
     description: "Add JSON linting configuration works correctly",
     type: "functional",
     source: "generated"
   }
   ```

3. **Augment with NFRs** (always):
   ```typescript
   // From story.nfrs.performance.maxResponseTime = "2s"
   {
     id: "AC-002",
     priority: "P0",
     description: "Response time < 2s",
     type: "non-functional",
     source: "nfr"
   }
   ```

### Step 3: Search for Implementation

**Keyword-based search:**
```typescript
// AC-001: JSON files validated
const keywords = ['json', 'lint', 'validation'];

// Search implementation files
await Grep('json', { glob: '**/*.ts', output_mode: 'files_with_matches' });
// → ['.jsonlintrc', 'src/linter.ts']

ac.implemented = implementationFiles.length > 0;
ac.evidence.implementation = implementationFiles;
```

### Step 4: Search for Tests

**Keyword + similarity scoring:**
```typescript
// Search test files
await Grep('json|lint', { glob: '**/__tests__/**/*.test.ts' });
// → ['src/__tests__/linter.test.ts']

// Read test file and extract test names
const tests = extractTestDescriptions(testFileContent);
// → [{ name: 'should validate JSON files', line: 10 }]

// Calculate similarity
const similarity = calculateSimilarity(ac.keywords, test.name);
// → 0.85 (85% match)

if (similarity >= 0.7) {
  ac.tested = true;
  ac.tests.push(testFile);
}
```

**Similarity calculation (Jaccard):**
```
AC keywords: {json, lint, validation}
Test name:   "should validate JSON files"
Test words:  {validate, json, files}

Intersection: {json} = 1
Union:        {json, lint, validation, validate, files} = 5

Similarity = 1/5 = 0.20 (below threshold, not matched)
```

**Why low?** Test uses "validate" (verb) vs "validation" (noun). Solution: Expand keywords with variants.

### Step 5: Calculate Per-AC Coverage

**From Jest coverage report:**
```typescript
// AC-001 linked to: src/linter.ts
const fileCoverage = coverageData['src/linter.ts'];

// Count covered statements
totalStatements = 50;
coveredStatements = 45;

ac.coverage = (45 / 50) * 100; // 90%
```

### Step 6: Identify Gaps

```typescript
const p0ACs = acs.filter(ac => ac.priority === 'P0');
const p0Tested = p0ACs.filter(ac => ac.tested);

if (p0Tested.length < p0ACs.length) {
  const gaps = p0ACs.filter(ac => !ac.tested);
  // → AC-002: Response time < 2s (not tested)

  for (const ac of gaps) {
    ac.gap = ac.implemented
      ? 'Implementation exists but no tests found'
      : 'Not implemented';
  }
}
```

### Step 8: Check Quality Gates

**Validate P0 coverage:**
```typescript
const config = {
  p0_threshold: 100,  // P0 must be 100% tested
  p1_threshold: 80,
  p2_threshold: 50,

}
```

## AC Generation Algorithm

```
if story.body contains "## Acceptance Criteria":
  parse ACs from markdown checklists
  → AC-001, AC-002, ... (from story)
else:
  generate functional AC from title
  → AC-001: "Title works correctly"

if story.nfrs exists:
  generate NFR-based ACs
  → AC-00X: "Response time < 2s"
  → AC-00Y: "Authentication via Cognito"

assign sequential IDs: AC-001, AC-002, AC-003...
extract keywords for each AC
return all ACs
```

## Test Linking Strategy

```
for each AC:
  1. Extract keywords (remove stop words)
  2. Search implementation files: Grep(keywords, glob: **/*.ts)
  3. Exclude test files: filter(!includes('__tests__'))
  4. Mark as implemented if files found

  5. Search test files: Grep(keywords, glob: **/__tests__/**/*.test.ts)
  6. Read each test file
  7. Extract describe/it blocks
  8. Calculate similarity(AC.keywords, test.name)
  9. If similarity >= 0.7: link AC to test

  10. Calculate coverage from Jest report (if available)
  11. Set AC.coverage, AC.tested, AC.evidence
```

## Module Inference

**For test stub generation:**
```
if story.context.relatedCode includes 'lambda/auth-edge':
  → module = 'lambda/auth-edge'
else if AC.keywords includes 'api' or 'endpoint':
  → module = 'lambda/referral'
else if AC.keywords includes 'auth' or 'cognito':
  → module = 'lambda/auth-edge'
else if AC.keywords includes 'infrastructure' or 'cdk':
  → module = 'infrastructure'
else:
  → module = 'src' (default)

testPath = `${module}/__tests__/ac-${id.toLowerCase()}.test.ts`
```

## Best Practices

1. **Always augment with NFRs** - Even if ACs exist in story body
2. **Use 70% similarity threshold** - Balances precision and recall
3. **Generate stubs for P0 only** - Avoid test bloat
4. **Infer module from context** - Use story.context.relatedCode first
5. **Block commits strictly** - P0 = 100%, no exceptions

## Troubleshooting

**Issue: ACs not detected in story body**
- Check: Heading format "## Acceptance Criteria" (case-sensitive)
- Fix: Use exact heading or rely on generation from title

**Issue: Test linking finds no matches**
- Check: Keywords extracted correctly (`ac.keywords`)
- Debug: Run Grep manually with keywords
- Adjust: Lower similarity threshold to 0.6

**Issue: Wrong module inferred**
- Check: story.context.relatedCode has correct paths
- Fix: Add more keywords to AC (e.g., "lambda", "infrastructure")

**Issue: Coverage shows 0% but tests exist**
- Check: Jest coverage report exists (run `npm test -- --coverage`)
- Fix: Link coverage report path in compliance config
